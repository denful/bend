<Aside icon="nix" title="Bending Nix Data" type="note">
Bend draws from Haskell profunctor optics, Scala's `Either`, the `adapt` primitive from [denful/nfx](https://github.com/denful/nfx) and parser-combinators.

The core idea is [Parse, Don't Validate](https://lexi-lambda.github.io/blog/2019/11/05/parse-don-t-validate/): a validator for non-empty lists that returns `true` is discarding the proof it just computed; instead if the function returns `left empty` or `right { head; tail; }`, the data structure *is* the proof.
Every lens either refines data as proof on `right` or returns invalid data on `left`.

Bend’s combinator names deliberately match parser-combinator vocabulary:
This is not cosmetic. The names signal that [Bend is a parser combinator library](/explanation/validation-is-parsing), one that operates on Nix data structures instead of character streams, and that happens to also support bidirectional writes.
</Aside>

### The [`adapt`](/explanation/adapt) primitive

Everything in Bend composes from a single combinator:

```nix
adapt from back refine lens
# from    — extract inner source from outer
# back    — write inner result back into outer
# refine  — refine focused value
# lens    — inner lens
```

### Either

`right` and `left`, both carry data not only booleans or error strings.

```nix
bend.right 42        # { right = 42; }
bend.left  "hello"   # { left  = "hello"; }

bend.mapR  (x: x + 1) (bend.right 5)               # { right = 6; }
bend.mapL  (x: x + 1) (bend.left  5)               # { left  = 6; }
bend.chain (x: bend.right (x * 2)) (bend.right 5)  # { right = 10; }
bend.swap  (bend.right 1)                           # { left  = 1; }
```

### Parse, Don't Validate

A validators return refined, structured data perserve validation proofs.

```nix
bend.nonEmpty.get [ 1 2 3 ]   # right { head = 1; tail = [ 2 3 ]; }
bend.nonEmpty.get [ ]         # left [ ]
```

### Composing lenses

`compose` threads two lenses. `pipe` composes a list left-to-right. Each step either refines or short-circuits with the point of failure.

```nix
let lens = bend.pipe [
  (bend.attr "name")
  bend.str
  (bend.satisfy (s: s != ""))
];
in
lens.get { name = "alice"; }   # right "alice"
lens.get { name = ""; }        # left ""
lens.get { name = 42; }        # left 42
lens.get { }                   # left { }
```

### Bidirectionality

Lenses write back through the same path they read. `set` returns `right` on success or `left` if the path cannot be reached.

```nix
let lens = bend.pipe [
  (bend.attr "config")
  (bend.attr "timeout")
];
in
lens.get { config = { timeout = 30; }; }
# right 30

lens.set { config = { timeout = 30; retry = 3; }; extra = true; } 60
# right { config = { timeout = 60; retry = 3; }; extra = true; }
```

`attr` returns `left` on `set` when the key is absent — you cannot write through a missing path.

### Schema validation

`record` validates each field and short-circuits on the first failure. `recordAll` gathers all failures.

```nix
(bend.record {
  name = bend.str;
  age  = bend.int;
}).get { name = "alice"; age = 30; extra = true; }
# right { name = "alice"; age = 30; }

(bend.recordAll {
  name = bend.str;
  age  = bend.int;
}).get { name = "alice"; age = "thirty"; }
# left {
#   name = right "alice";
#   age  = left { field = "age"; got = "thirty"; };
# }
```

### List parsing

`each` applies a lens to every element and returns per-element proof on failure. Bidirectional: validates on both `get` and `set`.

```nix
(bend.each bend.int).get [ 1 2 3 ]
# right [ 1 2 3 ]

(bend.each bend.int).get [ 1 "x" 3 ]
# left [ (right 1) (left "x") (right 3) ]

(bend.each bend.int).set [ 1 2 3 ] [ 10 20 30 ]
# right [ 10 20 30 ]

(bend.each bend.int).set [ 1 2 3 ] [ 10 "x" 30 ]
# left [ (right 10) (left "x") (right 30) ]
```

### Cardinality combinators

```nix
(bend.many  bend.int).get [ ]         # right []    — zero or more
(bend.some  bend.int).get [ ]         # left []     — one or more required
(bend.atLeast 2 bend.int).get [ 1 ]   # left [1]    — at least 2 required
(bend.exactly 3 bend.int).get [ 1 2 ] # left [1 2]  — exactly 3 required
```

### Error recovery with fallbacks

`alt` tries a second lens when the first fails. `recover` applies a function to the error.

```nix
let lens = bend.alt
  (bend.attr "preferred")
  (bend.attr "fallback");
in
lens.get { fallback = "value"; }  # right "value"

let lens = bend.recover
  (e: bend.right "default")
  (bend.attr "name");
in
lens.get { }  # right "default"
```

### Error labeling and validation

`label` replaces errors with custom messages. `ensure` adds post-parse validation. `annotate` tracks context.

```nix
let lens = bend.pipe [
  (bend.label "age field missing" (bend.attr "age"))
  (bend.ensure (n: n > 0) "age must be positive")
];
in
lens.get { }            # left "age field missing"
lens.get { age = -5 }   # left "age must be positive"
```

### Conditional and optional lenses

`when` and `unless` apply a lens only if a condition holds. `option` provides a default on failure.

```nix
let lens = bend.when (s: s ? admin) (bend.attr "admin");
in
lens.get { admin = true; }   # right true
lens.get { }                 # right { } — unchanged

let lens = bend.option "guest" (bend.attr "user");
in
lens.get { }  # right "guest" — default on missing
```

### Deep attribute access

`path` chains `attr` lenses for nested data. `index` accesses list elements by position.

```nix
let lens = bend.path [ "config" "server" "port" ];
in
lens.get { config = { server = { port = 8080; }; }; }
# right 8080

(bend.index 1).get [ "a" "b" "c" ]  # right "b"
(bend.index 5).get [ "a" "b" ]      # left [ "a" "b" ]
```

### Value transformation

`map` transforms a lens result. `bimap` transforms both left and right. `rmap`/`lmap` transform one side.

```nix
let lens = bend.map (x: x * 2) (bend.attr "count");
in
lens.get { count = 5 }  # right 10

let lens = bend.rmap (s: "got: ${s}") bend.str;
in
lens.get "hello"  # right "got: hello"
```
